# ScreenCast — трансляция экрана между двумя Android-телефонами

Простое приложение на Kivy: один телефон транслирует свой экран,
второй — принимает и показывает картинку. Передача идёт напрямую
по локальной Wi-Fi сети (оба телефона должны быть в одной сети).

## Как это работает

- **Приёмник**: поднимает TCP-сервер на порту 8765 и ждёт кадры.
- **Отправитель**: запрашивает у Android разрешение на запись экрана
  (системное окно "Начать запись/трансляцию?"), после этого начинает
  захватывать экран через `MediaProjection` + `VirtualDisplay` +
  `ImageReader`, сжимает каждый кадр в JPEG и шлёт по сокету на IP
  приёмника примерно 12 раз в секунду.

Это не видеозвонок и не сжатие видео (H.264) — это поток JPEG-картинок,
поэтому качество/плавность проще, чем в настоящих приложениях для
удалённого доступа, зато код короткий и понятный.

## Сборка APK

Сборка возможна только через Linux-инструментарий (python-for-android).
Если у тебя нет Linux-устройства, есть два варианта:

### Вариант A — GitHub Actions (без своего Linux, сборка в облаке)

1. Создай новый репозиторий на github.com и залей туда всю папку
   `screencast_app` (вместе с `.github/workflows/build.yml`).
2. Зайди во вкладку **Actions** репозитория — сборка запустится
   автоматически (или нажми "Run workflow" вручную).
3. Через несколько минут в этом же запуске появится артефакт
   **screencast-apk** — скачай его, внутри будет готовый `.apk`.
4. Перенеси `.apk` на телефон (через USB, облако, телеграм и т.д.)
   и установи (может понадобиться разрешить "установку из
   неизвестных источников").

### Вариант B — WSL2 на Windows

Сборка возможна на Linux или в WSL2 с установленным `buildozer`.

```bash
pip install buildozer cython
sudo apt install -y git zip unzip openjdk-17-jdk python3-pip autoconf \
    libtool pkg-config zlib1g-dev libncurses5-dev libncursesw5-dev \
    libtinfo5 cmake libffi-dev libssl-dev

cd screencast_app
buildozer -v android debug
```

Готовый APK появится в `bin/screencast-0.1-armeabi-v7a-debug.apk`
(или похожее имя). Установи его на оба телефона.

## Использование

1. Подключи оба телефона к одной Wi-Fi сети.
2. На телефоне-приёмнике: открой приложение → "Я приёмник" →
   посмотри свой IP-адрес (Настройки → Wi-Fi → нажать на текущую сеть).
3. На телефоне-отправителе: "Я отправитель" → введи IP приёмника →
   "Начать трансляцию экрана" → подтверди системный диалог о записи экрана.
4. На приёмнике должна появиться картинка с экрана отправителя.

## Важные оговорки

- **Android 10+ (API 29+)**: формально Google требует, чтобы
  `MediaProjection` запускался вместе с foreground-сервисом типа
  `mediaProjection`. В этом каркасе такого отдельного сервиса нет —
  на многих устройствах и версиях всё равно работает, но если
  система выбросит `SecurityException`, значит именно это и есть
  причина. Добавление полноценного foreground-сервиса через pyjnius
  — следующий шаг, если понадобится (могу помочь с этим отдельно).
- Приложение шлёт кадры без шифрования — только для домашней/локальной
  сети, не для интернета.
- Скорость и качество (12 fps, JPEG качество 40, макс. сторона 720px)
  настраиваются константами в начале `main.py` (`TARGET_FPS`,
  `JPEG_QUALITY`, `MAX_CAPTURE_SIDE`).
